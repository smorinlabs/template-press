# `init/` ‚Äî blueprint self-setup system

Self-contained tooling that guides a developer through rebranding this repo
after it has been instantiated from the `py-launch-blueprint` template. Lives
entirely under `init/` so the system can be removed in one step once it has
done its job.

| Component | Type | Purpose |
|---|---|---|
| **`guard.sh`** | POSIX shell, two modes | Two-tier setup guard. Tier 1: universal stderr discovery warning on every recipe run (`warn` mode). Tier 2: hard block on the risk subset of recipes (`block` mode). |
| **`init.py`** | PEP 723 script (`uv run`) | Interactive (or `--config`-driven) walkthrough that deterministically rewrites blueprint identity into the new project's values. Strict one-shot (refuses if marker exists, unless `--force`). |
| **`init_doctor.py`** | PEP 723 script (`uv run`) | Audits migration completeness + environment readiness. `--fix` only touches the safe environment class (`uv sync`, hook install). Adds a `post_init/` check class that verifies workflow-file locations match marker state, ci.yml codecov gate is present when expected, and OIDC verification timestamps. |
| **`post_init.py`** | PEP 723 script (`uv run`) | Interactive walkthrough that runs after `init`. Configures Publishing (PyPI + TestPyPI + release-please), Codecov, ReadTheDocs. "No" responses move workflow files to `.github/workflows.disabled/`. `--skip-remote` for local-only edits; `--status` to print current state. Records decisions in the marker's `[post_init]` table. |
| **`manifest.toml`** | TOML | Single source of truth for migration scope ‚Äî `[[replace]]` / `[[rename]]` / `[[remove]]` / `[[regenerate]]` blocks. Generated by `discover.py`. |
| **`common.py`** | Python module (stdlib) | Identity ground truth, manifest schema, answer validators, origin URL parser. Imported by every other component. |
| **`_engine.py`** | Python module | The rewrite engine ‚Äî applies the manifest given user-supplied answers. Order: **remove ‚Üí replace ‚Üí rename**, then **regenerate**. |
| **`_rewriters.py`** | Python module | Per-file structured rewriters (empty registry today; extension seam for files where text-mode replacement isn't safe). |
| **`discover.py`** | PEP 723 script | One-off Phase-0 utility ‚Äî scans the repo for identity-string occurrences and emits a draft `manifest.toml`. |
| **`ci/`** | CI helpers | `check_no_marker.sh`, `check_guard_wiring.py`, `check_manifest_drift.py` ‚Äî all callable from `init_doctor` and the `blueprint-guard` workflow. |
| **`tests/`** | pytest | Five `¬ß4.7` instantiation-mode tests plus manifest and doctor coverage. |

## Usage

```bash
just init                              # interactive walkthrough
just init --config answers.toml        # headless / CI
just init --dry-run                    # print the plan, don't write
just init-doctor                       # audit migration + environment
just init-doctor --fix                 # remediate safe environment issues
just post-init                         # configure publishing/Codecov/RTD
just post-init --status                # show post-init state, no changes
just post-init --skip-remote           # local-only edits, defer gh/PyPI
```

## Quick test

```bash
uv run pytest init/tests/ --override-ini="addopts="
```

## Removing the system

After the project has been initialized and the marker committed, the system
self-silences (guard skips when the marker is present). Remove it entirely
with:

```bash
just init --prune
```

‚Ä¶then manually delete `_blueprint_notice`, `_guard`, `init`, `init-doctor`
from `Justfile` and the `init/.blueprint-contributor` line from `.gitignore`
(those edits are out of scope for automated removal).

## Design

See `init/init-spec.md` (in this repo) for the full design rationale,
including the two-tier guard architecture, the manifest schema, and the
five ¬ß4.7 instantiation modes (template button, `gh` CLI, clone-reinit,
fork, ZIP) that the tests pin as a regression contract.
